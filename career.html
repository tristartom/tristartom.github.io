Project: Data-Intensive Cryptographic Engineering on Hardware Enclaves

Preliminary Results

  Thrust 1: Oblivious Data Analytics with Systems-level Efficiency

  1. "Cache-Miss Oblivious Data Shuffling on Hardware Enclave",
    Ju Chen, Yuzhe (Richard) Tang, Hao Zhou.
    SysTex 2017 at ACM SOSP (Workshop paper)
    • The work is to build a data shuffling capability on Intel SGX platform in a unique way that is both strongly secure (or oblivious memory access) and efficient. The key idea to lower performance overhead without sacrificing strong security is the notion of cache-miss obliviousness. Upon implementation, the cache-miss obliviousness is realized by leveraging Intel TSX features.
  2. "Towards Building Practical And Secure Multi-Party Databases",
    Yuzhe Tang, Wenqing Zhuang.
    IEEE SecDev 2016. [pdf]
    • The work is to build a database system on top of multiple private-data sources. A user can submit a regular SQL query to our database which evaluates the query among data sources while preserving the privacy of each data source. Comparing existing work on applied multi-party computations (MPC), our work is distinct in building real MPC-based systems.
  Thrust 2: Authenticated Data-Storage Systems
  1. "Lightweight Authentication of Freshness in Outsourced Key-Value Stores"
    Yuzhe Tang, Ting Wang, Ling Liu, Xin Hu, Jiyong Jang
    ACSAC 2014 (AR<19.9%), [pdf, ICDE demo: pdf]
    • In this work, we consider a client outsourcing a key-value store to an untrusted cloud wants to ensure data returned from the cloud-side storage is up-to-date and fresh. The freshness verification essentially requires to efficiently link the data write signed in the past to the write signed recently. We propose a Merkle tree structure augmented with Bloom filters which improves write performance while enabling freshness authentication. The proposed scheme is implemented in Apache HBase.
  2. "Deferred Lightweight Indexing for Log-Structured Key-Value Stores"
    Yuzhe Tang, Arun Iyengar, Wei Tan, Liana Fong, Ling Liu, Balaji Palanisamy
    CCGrid 2015, (AR<25.7%), [pdf, src]
    ★ Best Paper Award [link]
    • This work supports secondary index on a "NoSQL" key-value store, and tackles the optimization of write performance. Inspire by the Log-structured Merge (LSM) design that optimizes the write performance of primary index, our key idea is to apply LSM tree to the secondary index design. The distinct design comes from the mapping of the LSM design to the underlying sharded index-base partitions. We proposed a mechanism to defer index maintenance without sacrificing the performance of serving online reads and writes. We implemented the proposed scheme on Apache HBase without modifying its internal codebase, and open-source the code on github.
  3. "Log-Structured Authenticated Data Structure with Minimal Trust (Or When LSM tree meets ADS)"
    Yuzhe Tang, Ju Chen.
    IACR Report 2016, [pdf, Related theoretic facts: pdf]
    • The work proposes a novel approach to building authenticated cloud storage systems by combining Authentication Data Structure (ADS) and Log-Structured Merge Trees (LSM trees). ADS is a security protocol used to ensure data authenticity in an outsourced key-value store. LSM tree is a data structure used to build cloud key-value stores with write-optimized performance. This work is the first to combine these two structures together and builds an overall storage system. The end result is that data can be authenticated without (significantly) slowing down the write performance in a log-structured cloud storage system.
