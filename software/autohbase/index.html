Software AutoHBase
Yuzhe Tang

What if you have to repeatedly type system administration commands thousands of times to get a convincing experiment result? What if you need to wait for a distributed system to fully stop, while it may hang silently?

Introduction: Setting up and running a distributed system (e.g. for conducting experiments) is a daunting and dirty task. Automation is crucial. AutoHBase provides an automatic deployment and administration functionality for HBase in a distributed cluster. By simply copy-and-pasting the bash command sequence below, one can easily set up an HBase cluster automatically or within one click.

AutoHBase is currently targeted on Unix/Linux users and is implemented by Bash as a wrapper around original HBase scripts in the 0.94.2 release.

ReadMe: Instructions to setup a functioning HBase cluster without hassles.

Prerequisite:

  • Downloading the software at [link].
  • Have passphrase-less ssh setup between all machines where the HBase cluster is deployed and the operating machine where the software is downloaded. One can use my QuickSSH software to quickly set up passphrase-less SSH in a snap.
  • Have java (6 or above) setup on all deployment machine.

Deploy an HBase cluster:

  • Configure config_vfs.sh with a desired destination directory, say $destination_dir. For example, destination_dir=/tmp/hbase, then in file config_vfs.properties, do the following configuration:
    • hbase_all_deployedProjDir=/tmp/hbase/hbase/
    • hbase_master_deployedProjAdminDir=/tmp/hbase/hbaseAdmin/
    • unzip autohbase.zip
    • cd autohbase/
    • destination_dir=/tmp/hbase
    • mkdir -p $destination_dir
    • ./tt_sh/apply_config.sh
    • ./tt_sh/deploy.sh

Administrating the HBase cluster:

  • Start the HBase cluster:
    • cd $destination_dir
    • sudo su
    • cd hbaseAdmin/editable
    • ./tt_sh/startHbase_verified.sh 10 no
    • At the end of the terminal printout, following text is expected to show, which indicates a success of launching an HBase cluster.
    • @localhost
    • 14214 HQuorumPeer
    • 14280 HMaster
    • 14684 Jps
    • 14469 HRegionServer
  • Enter the HBase shell:
    • cd $destination_dir
    • ./tt_sh/shellHbase.sh
  • Stop the HBase cluster:
    • cd $destination_dir
    • ./tt_sh/shellHbase.sh

Advanced features in config_vfs.sh (to be fully documented):

  • Load a prematerialized dataset:
    • ./reload_hbase.sh
    • ./tt_sh/startHbase_verified.sh 10 yes
  • Distributed setup:
    • Modify field hbase_regionserver_hostnames to a list of nodes in config_vfs.sh.
  • Setup of an HDFS cluster as an HBase substrate:
    1. Modify field hdfs_* in config_vfs.sh.
