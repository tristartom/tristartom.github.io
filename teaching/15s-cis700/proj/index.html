Programming project

Policy

Programming project can be done individually or collaboratively (by two students). Following is a list of sample topics. Talk to me about details.

  • Analysis in distributed social networks
  • Multi-party similarity computation
  • SGX programming
  • Verifiable sorting
  • Request trace analysis in cloud
  • NoSQL on SSD (reading [link])
  • NoSQL with tail latency (reading [link])
  • JVM-based profiling in Hadoop

Or you can start by exploring various open-source cloudy software [link]. The only requriement is that the proposed project must be implementable using open-source software.

Project proposal (Jan. 30)

Submit a one-page report, summarizing the context, movitation and goal of the proposed project.

Mid-term report (Feb. 27)

Do a presentation, talk about the value, design and implementation of your project.

Final demo (April. 17)

Show the working of your project.

Final report (May. 1)

Formal writeup about everything of your project.
